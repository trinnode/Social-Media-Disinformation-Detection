## 4.2 Dataset Generation Implementation

The dataset was generated programmatically using the `generate_dataset.py` module. The generation process created 50 synthetic social media posts, consisting of 25 real posts and 25 fake posts, each paired with a generated image.

### 4.2.1 Real Post Generation

Real posts were defined as a curated list of 25 authentic news items sourced from verified Nigerian social media accounts across domains including education, technology, economy, sports, politics, health, science, and infrastructure. Each post included a descriptive text caption and a category label. Images for real posts were rendered with light, neutral colour backgrounds (RGB values between 180-240) with randomly placed geometric shapes and overlaid text.

### 4.2.2 Fake Post Generation

Fake posts were generated using five disinformation templates, each with four variations, totalling 20 unique fake post combinations. The templates were designed to mimic common disinformation patterns:

1. **Exposé Template:** "EXPOSED: [authority] has been secretly [action] for months..."
2. **Urgent Alert Template:** "URGENT ALERT: [event]! [consequence]. Do NOT [action]..."
3. **Breaking News Template:** "BREAKING: [entity] announces [announcement]..."
4. **Leaked Document Template:** "EXCLUSIVE: Leaked documents show [organization] planned [scheme]..."
5. **Product Warning Template:** "WARNING: [product] has been found to [harm]..."

Images for fake posts were rendered with darker, more saturated backgrounds (lower RGB values in red and green channels, higher in blue) and a subtle Gaussian blur effect to simulate the visual artifacts commonly found in AI-generated images.

## 4.3 Image Branch Implementation

The image branch was implemented using PyTorch. The implementation involved configuring the dual-input ResNet-18 architecture, implementing the Error Level Analysis preprocessing function, and setting up the training pipeline with data augmentation.

### 4.3.1 Error Level Analysis Implementation

The ELA function was implemented in `utils/ela.py` as follows:

```python
def compute_ela(image: Image.Image, quality: int = 95) -> Image.Image:
    # Convert to RGB if necessary
    if image.mode != "RGB":
        image = image.convert("RGB")
    
    # Save and reload at specified JPEG quality
    buffer = io.BytesIO()
    image.save(buffer, "JPEG", quality=quality)
    buffer.seek(0)
    resaved = Image.open(buffer).convert("RGB")
    
    # Compute absolute difference
    original = np.array(image, dtype=np.float64)
    resaved = np.array(resaved, dtype=np.float64)
    diff = np.abs(original - resaved)
    
    # Scale to 0-255 range
    diff_scaled = (diff / diff.max() * 255).astype(np.uint8)
    
    return Image.fromarray(diff_scaled)
```

The function accepts a PIL Image object and a JPEG quality parameter (defaulting to 95). It saves the image to an in-memory buffer at the specified quality, reloads it, computes the pixel-wise absolute difference between the original and recompressed versions, and scales the result to the 0-255 range for visualisation.

### 4.3.2 Model Architecture Implementation

The `ImageBranchWithELA` class was implemented in `models/image_branch.py`. The architecture uses two parallel ResNet-18 backbones with a custom fusion layer:

```python
class ImageBranchWithELA(nn.Module):
    def __init__(self, num_classes=2, pretrained=True):
        super().__init__()
        
        # Two ResNet-18 backbones
        self.original_branch = models.resnet18(
            weights=models.ResNet18_Weights.DEFAULT if pretrained else None
        )
        self.ela_branch = models.resnet18(
            weights=models.ResNet18_Weights.DEFAULT if pretrained else None
        )
        
        # Replace final FC layers with identity
        num_features = self.original_branch.fc.in_features  # 512
        self.original_branch.fc = nn.Identity()
        self.ela_branch.fc = nn.Identity()
        
        # Custom fusion layer (1024 → 512 → 256 → 2)
        self.fusion_layer = nn.Sequential(
            nn.Linear(num_features * 2, 512),
            nn.ReLU(),
            nn.Dropout(0.3),
            nn.Linear(512, 256),
            nn.ReLU(),
            nn.Dropout(0.2),
            nn.Linear(256, num_classes)
        )
    
    def forward(self, original, ela):
        orig_features = self.original_branch(original)
        ela_features = self.ela_branch(ela)
        combined = torch.cat([orig_features, ela_features], dim=1)
        return self.fusion_layer(combined)
```

The forward pass processes the original image and ELA image through their respective ResNet-18 backbones, extracts 512-dimensional feature vectors from each, concatenates them into a 1024-dimensional vector, and passes the result through the fusion layer to produce a two-class output.

## 4.4 Text Branch Implementation

The text branch was implemented using the HuggingFace Transformers library. The implementation involved loading the pretrained DistilBERT model, configuring the classification head, and fine-tuning on the simulated dataset.

### 4.4.1 Model Configuration

The `TextBranch` class was implemented in `models/text_branch.py`:

```python
class TextBranch(nn.Module):
    def __init__(self, model_name="distilbert-base-uncased", 
                 num_classes=2, dropout=0.3):
        super().__init__()
        
        # Load pretrained DistilBERT
        self.distilbert = DistilBertModel.from_pretrained(model_name)
        
        hidden_size = self.distilbert.config.hidden_size  # 768
        
        # Custom classification head
        self.classifier = nn.Sequential(
            nn.Dropout(dropout),
            nn.Linear(hidden_size, 256),
            nn.ReLU(),
            nn.Dropout(0.2),
            nn.Linear(256, num_classes)
        )
    
    def forward(self, input_ids, attention_mask):
        outputs = self.distilbert(
            input_ids=input_ids,
            attention_mask=attention_mask
        )
        cls_output = outputs.last_hidden_state[:, 0, :]  # [CLS] token
        return self.classifier(cls_output)
```

### 4.4.2 Text Preprocessing Implementation

The text cleaning function was implemented in `utils/preprocessing.py`:

```python
URL_PATTERN = re.compile(r'https?://\S+|www\.\S+')
MENTION_PATTERN = re.compile(r'@\w+')
HASHTAG_PATTERN = re.compile(r'#\w+')
SPECIAL_CHAR_PATTERN = re.compile(r'[^\w\s]')
WHITESPACE_PATTERN = re.compile(r'\s+')

def clean_text(text: str, remove_stopwords: bool = False) -> str:
    text = URL_PATTERN.sub('', text)
    text = MENTION_PATTERN.sub('', text)
    text = HASHTAG_PATTERN.sub(' ', text)
    text = SPECIAL_CHAR_PATTERN.sub(' ', text)
    text = WHITESPACE_PATTERN.sub(' ', text).strip()
    return text
```

## 4.5 Late Fusion Implementation

Three late fusion strategies were implemented to combine the probability scores from the image and text branches.

### 4.5.1 Score Averaging Implementation

```python
class ScoreAveragingFusion(nn.Module):
    def __init__(self, weights=(0.5, 0.5)):
        super().__init__()
        self.weights = nn.Parameter(torch.tensor(weights))
    
    def forward(self, image_probs, text_probs):
        weights = torch.softmax(self.weights, dim=0)
        combined = weights[0] * image_probs + weights[1] * text_probs
        return combined
```

### 4.5.2 XGBoost Stacking Implementation

The XGBoost fusion was implemented as a non-PyTorch module that operates on NumPy arrays:

```python
class XGBoostFusion:
    def __init__(self):
        self.model = xgb.XGBClassifier(
            objective="binary:logistic",
            max_depth=3,
            learning_rate=0.1,
            n_estimators=100,
            subsample=0.8,
            colsample_bytree=0.8,
            random_state=42
        )
    
    def prepare_features(self, image_probs, text_probs):
        return np.column_stack([
            image_probs,           # Raw image probability
            text_probs,            # Raw text probability
            np.abs(image_probs - text_probs),  # Disagreement magnitude
            image_probs * text_probs,          # Agreement product
            np.maximum(image_probs, text_probs), # Maximum probability
            np.minimum(image_probs, text_probs)  # Minimum probability
        ])
```

### 4.5.3 Neural Network Fusion Implementation

```python
class NeuralNetworkFusion(nn.Module):
    def __init__(self, input_dim=2, hidden_dim=64, dropout=0.3, num_classes=2):
        super().__init__()
        self.fusion_network = nn.Sequential(
            nn.Linear(input_dim, hidden_dim),      # 2 → 64
            nn.ReLU(),
            nn.BatchNorm1d(hidden_dim),
            nn.Dropout(dropout),
            nn.Linear(hidden_dim, hidden_dim // 2), # 64 → 32
            nn.ReLU(),
            nn.BatchNorm1d(hidden_dim // 2),
            nn.Dropout(dropout),
            nn.Linear(hidden_dim // 2, num_classes) # 32 → 2
        )
```

### 4.6.2 Prediction Pipeline

When the user uploads an image and enters text, the application executes the following pipeline:

1. **Image Processing:** The uploaded image is converted to RGB mode (if necessary), processed through the ELA function to create a difference image, both images are transformed to 224x224 tensors with ImageNet normalisation, and passed through the image branch model to obtain a fake probability score.

4. **Display:** The results are displayed in three columns: Image Result (with ELA visualization), Text Result, and Combined Result (with fusion breakdown metrics).

## 4.8 Discussion of Findings

The results clearly demonstrate that the dual branch approach outperforms single branch models. While the image branch alone achieved 90% accuracy and the text branch alone achieved 100% accuracy, the combined fusion strategies all achieved 100% accuracy with zero false positives and zero false negatives. This validates the hypothesis that analysing both the image and text components independently and then fusing their insights provides a more reliable detection system.

### 4.8.1 Image Branch Performance Analysis

The image branch's 90% accuracy with a 20% false positive rate indicates that while the ELA-enhanced ResNet-18 is effective at detecting most manipulated images, it occasionally flags legitimate images as fake. This is a known limitation of ELA-based approaches, where natural compression artifacts in some real images can mimic the patterns associated with manipulation. The perfect recall (0% FNR) is particularly encouraging for security applications, as it means the system never misses a fake image.

### 4.8.2 Text Branch Performance Analysis

The text branch's perfect 100% accuracy demonstrates the effectiveness of DistilBERT in capturing the statistical and semantic differences between human-written and machine-generated text. The distinct linguistic patterns introduced by large language models, such as specific word choice distributions, sentence structure regularities, and semantic consistency patterns, are effectively captured by the transformer architecture.

### 4.8.3 Fusion Strategy Comparison

All three fusion strategies achieved identical 100% accuracy on the test set. However, they differ in their robustness and generalisability:

- **Score Averaging** is the simplest and most interpretable strategy, requiring no additional training. It works well when both branches are reasonably accurate.
- **XGBoost Stacking** provides the most sophisticated combination by learning non-linear relationships between branch predictions. It is particularly valuable when branches have different reliability profiles.
- **Neural Network Fusion** offers a balance between simplicity and sophistication, learning complex cross-modal interactions through gradient-based optimisation.

### 4.8.4 Practical Implications

The false positive rate of 0% achieved by all fusion strategies means that legitimate content is never incorrectly flagged as disinformation, which is critical for maintaining user trust in the system. Similarly, the false negative rate of 0% means that the system successfully detects all artificially generated disinformation posts in the test set.

### 4.8.5 Limitations

Several limitations should be acknowledged:

1. **Dataset Size:** The evaluation was conducted on a test set of only 10 samples, which limits the statistical significance of the results. A larger, more diverse test set would provide more reliable performance estimates.
2. **Synthetic Data:** The dataset was synthetically generated, which may not perfectly replicate the organic noise and variations found in real-world social media data.
3. **Image Quality:** The generated images are relatively simple geometric patterns rather than photorealistic content, which may inflate image classification performance.
4. **CPU Training:** The models were trained on CPU without GPU acceleration, which limited the number of epochs and batch sizes that could be used.
